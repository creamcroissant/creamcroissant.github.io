使用 acme.sh 申请 IP 证书并部署到 Caddy

John Doe
2026-03-31
运维
Caddy, IP证书, Let's Encrypt, SSL, acme.sh

本文记录一套可复用流程：用 acme.sh 申请 Let’s Encrypt IP 证书，并在 Caddy 中加载使用。
适合没有域名、直接用公网 IP 提供 HTTPS 的场景。

0. 前置说明

  • 系统：Debian/Ubuntu（其他 Linux 同理）
  • ACME 客户端：acme.sh
  • Web 服务：Caddy
  • 验证方式：http-01（80 端口必须公网可达）
  • 证书类型：ECC（--ecc）

8. 常见问题速查

Q1：invalidContact + example.com

A：账户邮箱使用了示例域名，改成真实邮箱或不传邮箱。

Q2：Connection refused（验证失败）

A：CA 连不上 http://IP/.well-known/...，检查 80 端口监听/防火墙/安全组。

Q3：tcp port 80 is already used by caddy

A：你在 standalone 模式续期，但 Caddy 占用了 80；先停 Caddy 再续。

Q4：caddy.service is not active, cannot reload

A：你执行 reload 时服务本身没运行，先 systemctl start caddy。

9. 总结

完整链路可概括为：

  1. acme.sh 注册正式账户
  2. --issue 申请 IP 证书（推荐先测试环境验证流程）
  3. --install-cert 安装到 /etc/ssl/... 固定路径
  4. Caddy tls fullchain key 手动加载
  5. 做好自动续期（特别是 shortlived + standalone 的 80 端口冲突）

这套方案的关键不是“签发一次成功”，而是续期链路长期稳定。
